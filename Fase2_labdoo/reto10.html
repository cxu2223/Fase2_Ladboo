IMPLEMENTACIÓ DE SERVIDORS LINUX I WINDOWS

L'encàrrec consisteix en

Aquest encàrrec implica la implementació d'un nou conjunt de servidors per optimitzar la infraestructura de xarxa actual d'una escola amb una topologia heretada. Amb l'objectiu de millorar l'eficiència i la confiabilitat, es proposa l'ús de diversos servidors clau

  • El servidor DHCP facilitarà l'assignació dinàmica d'adreces IP, assegurant una gestió efectiva de la xarxa.
  • El servidor DNS, es millorarà la gestió dels noms de domini, essencial per a una navegació web eficient.
  • El servidor FTP permetrà la compartició d'arxius de manera segura i ràpida.
  • La implementació d'un servidor proxy millorarà la seguretat de la xarxa, filtrant i gestionant l'accés a Internet.
  • El servei de correu electrònic simplificarà la comunicació interna, oferint una plataforma eficaç per a la interacció entre membres de la comunitat educativa.

Aquest conjunt de millores té com a objectiu proporcionar una infraestructura de xarxa més robusta i adaptada a les creixents demandes de la institució educativa. A més, cal destacar que la implementació d'aquests servidors serà totalment customitzable i es realitzarà sota el criteri dels professionals encarregats de la infraestructura informàtica. Aquesta flexibilitat permetrà als responsables de la xarxa adaptar els nous servidors a les necessitats específiques de la institució educativa, tenint en compte els requeriments particulars i les consideracions pròpies de la topologia de la xarxa heretada. Això assegurarà una implementació personalitzada, optant per les configuracions que millor s'ajustin als objectius i requisits específics de la nostra escola. Aquest enfocament col·laboratiu amb els professionals garantirà una transició suau i efectiva cap a una infraestructura de xarxa més moderna i eficient.

DHCP

El DHCP (Dynamic Host Configuration Protocol) és un protocol de xarxa que assigna automàticament adreces IP i altres paràmetres de configuració a dispositius connectats a la xarxa, com ara ordinadors o telèfons intel·ligents. Això evita la necessitat de configurar manualment cada dispositiu amb una adreça IP, simplificant la gestió de la xarxa.
El nostre servei DHCP està configurat de la següent manera:

  • La nostra ID de xarxa és: 10.0.2.0/24
  • Les primeres 10 IP (10.0.2.1 - 10.0.2.11) queden excluides ja que les reservem per els professors, posteriorment ens servirà per diferenciar entre les restriccions dels alumnes i dels professors.
  • Les IP es renovaran cada tres dies.

Ubuntu Server

En aquest cas hem creat 5 subxarxes per dividir les 5 classes lògicament. Per al nostre Ubuntu Server haurem de fer diverses configuracions prèvies, primer ens assegurarem de tenir el programari actualitzat amb les següents comandes:

sudo apt update

sudo apt upgrade

També instal·larem una interfície gràfica per facilitar-nos el treball.

sudo apt install –no-install-recommends lubuntu-desktop

Un cop instalada la interfície gràfica, reiniciarem el servidor per començar a utilitzar-la.

sudo reboot

La última preparació prèvia serà configurar una IP estàtica al servidor, per això haurem de modificar el següent arxiu:

sudo nano /etc/netplan/00-installer-config.yaml

L'arxiu haurà de quedar així:

Un cop configurada la nostra IP estàtica haurem d'aplicar els canvis, és crucial que no surti cap error a l'hora d'executar la comanda, de lo contrari no podrem avançar al següent pas.

sudo netplan apply

Per a configurar DHCP a Ubuntu Server el primer que haurem de fer serà instal·lar el paquet isc-dhcp-server amb la següent comanda:

sudo apt-get install isc-dhcp-server

Desprès d'haver instal·lat el paquet, apagarem el nostre adaptador de xarxa per fer tota la configuració.

sudo ifconfig enp0s3 down

Per configurar les cinc subxarxes haurem de modificar el següent arxiu i afegir les següents lineas:

sudo nano /etc/dhcp/dhcpd.conf

Ara per indicar la interfície on volem que funcioni el DHCP haurem de modificar el següent arxiu i modificarem la següent linea:

sudo nano /etc/default/isc-dhcp-server

Un cop acabada la configuració engegarem el nostre adaptador de xarxa.

sudo ifconfig enp0s3 up

Per aplicar tots els canvis reiniciarem el servei DHCP.

sudo systemctl restart isc-dhcp-server

Per acabar, comprovarem que no tinguem cap error i el nostre servei estigui correctament engegat.

sudo systemctl status isc-dhcp-server

Per últim, verificarem que el nostre servei DHCP estigui donant les IP correctes des d'un client. El nostre client haurà de tenir la IP per DHCP i haurà de tenir una IP dins de les nostres subxarxes.

DNS

El DNS (Domain Name System) és un sistema que converteix els noms de domini, com ara "google.com", en adreces IP, que són números que les màquines utilitzen per identificar-se i comunicar-se entre si a la xarxa d'Internet. Bàsicament, funciona com una agenda telefònica per a la web, permetent als usuaris accedir als llocs web utilitzant noms de domini fàcils de recordar en lloc d'haver de recordar les seves adreces IP numèriques. El nostre servei DNS està configurat amb el següent domini:

  • Windows: grupo1labdoo.org
  • Ubuntu: grup1labdoo.org

Amb els següents hosts:

  • www
  • ftp
  • mail

Ubuntu Server

  • És important que s'hagin seguit les instruccions de com mantenir el programari actualitzat i com configurar una IP estàtica per al servidor. Tot això explicat a l'apartat DHCP.

Per començar haurem d'instal·lar els paquets necessaris.

sudo apt-get install bind9 bind9utils bind9-doc

Un cop instal·lats, modificarem el següent arxiu i haurà de quedar com s'aprecia a la captura de pantalla:

sudo nano /etc/bind/named.conf.options

Ara, modificarem aquest altre arxiu i haurà de quedar com s'aprecia a la captura de pantalla:

sudo nano /etc/default/named

Ara, comprovarem que les modificacions que hem fet siguin correctes.

sudo named-checkconf

Passarem a modificar el següent arxiu de manera que quedi como a la captura de pantalla:

sudo nano /etc/bind/named.conf.local

El següent que farem serà crear la zona directa i la zona inversa, haurem de crear una carpeta anomenada zones al directori /etc/bind.

sudo mkdir zones

Dins de la carpeta zones crearem un primer arxiu anomenat db.[EL VOSTRE DOMINI], en el nostre cas s'anomenará db.grup1labdoo.orgi el deixarem com s'aprecia a la captura. Aquest arxiu serà la nostra zona directa.

sudo nano db.grup1labdoo.org

Dins de la carpeta zones crearem un segon arxiu anomenat db.[LA VOSTRA IP INVERSA], en el nostre cas s'anomenará db.2.0.10i el deixarem com s'aprecia a la captura. Aquest arxiu serà la nostra zona inversa.

sudo nano db.2.0.10

Un cop creats els nostres arxius, aplicarem els canvis realitzats.

sudo named-checkzone grup1labdoodomain.org db.grup1labdooorg && db.02.0.10.in-addr.arpa db.02.0.10

Per últim reiniciarem el servei bind.

sudo systemctl restart bind9

Per comprovar que está en funcionament utilitzarem la comanda nslookup a un client.

nslookup www.grup1labdoo.org

FTP

FTP significa "File Transfer Protocol" (Protocol de Transferència de Fitxers). És un protocol de xarxa utilitzat per transferir fitxers entre ordinadors a través d'Internet o una xarxa local. Permet la transferència de fitxers entre un client (ordinador de l'usuari) i un servidor (ordinador remot) de manera eficient i segura. Els usuaris poden utilitzar clients FTP per connectar-se a servidors FTP i transferir fitxers, com ara documents, imatges o arxius de programari, entre els dos dispositius. Per al nostre FTP comptarem amb la mateixa estructura d'usuaris explicada a l'apartat CREACIÓ D'USUARIS I GESTIÓ DE PERMISOS I POLÍTIQUES I DIRECTIVES i amb una estructura de permisos similar.
